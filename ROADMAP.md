# Roadmap

## Near Term

- shrink repository footprint by moving large generated datasets out of Git
- keep a compact sample dataset for fast clone and fast demo
- add a small automated smoke test for planner execution
- publish a short demo video or GIF for the README
- clean up staging material that should live in a separate repo or branch

## Medium Term

- add reproducible benchmark scenarios for Bosphorus, Marmara, and Genoa corridors
- document supported input schemas for land, TSS, and bathymetry layers
- add release artifacts for sample datasets and reference outputs
- improve CLI help and example coverage
- add regression checks for route length, feasibility, and constraint handling

## Longer Term

- separate research experiments from stable planner components
- publish performance benchmarks and comparison scenarios
- document architecture and algorithm tradeoffs in more detail
- package reusable planner components for downstream applications
